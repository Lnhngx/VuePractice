什麼是Vue
1.構建用戶介面 
--用vue往html頁面填充數據，非常方便
2.框架
--框架是一套現成的解決方案，程序員只能遵守框架的規範，去編寫自己的功能
--要學習vue，就是在學習vue框架中的規定用法
--vue的指令，組件（是對UI結構的復用），路由，Vuex，vue組件庫
VUE的兩個特性
1.數據驅動視圖
--數據的變化會驅動視圖的更新
--好處：程序員只管把數據維護好，那麼頁面結構會被vue自動渲染出來
2.雙向數據綁定
  在網頁中，form表單負責採集數據，Ajax負責提交數據
  --js數據的變化，會被自動渲染到業面上
  --頁面上表單採集的數據發生變化的時候，會被vue自動獲取，並更新到js數據中
注意:數據驅動視圖和雙向數據綁定的底層原理是MVVM（Mode數據源、View視圖、ViewModel就是Vue的實例）

vue指令
1.內容渲染指令
 1.v-text指令的缺點：會覆蓋員元素內部原有的內容
 2.{{ }}插值表達式：在實際開發中應用最多，只是內容的佔位符，不會覆蓋原有的內容
 3.v-html指令的作用：可以把帶有標籤的字符串，渲染成真正的HTML內容
2.屬性綁定指令
注意：插值表達式只能用在元素的內容節點中，不能用在元素的屬性節點中
--在vue中，可以使用v-bind:指令，為元素的屬性動態綁定值
可簡寫成:
--在使用v-bind屬性綁定期間，如果綁定內容需要進行動態拼接，則字符串需要包裹單引號 例如<div :title="'box'+index">這是一個DIV</div>
3.事件綁定
 1.v-on 簡寫是@
 2.語法格式為：
 <button @click="add"></button>
 methods:{
   add(){
     //如果在方法中要修改data的數據，可以通過this
     this.count +=1
   }
 }
 3.$event的應用場景：如果默認的事件對象e被覆蓋，則可以手動傳第一個$event，例如
<button @click="add(3, $event)"></button>
 methods:{
   add(n,e){
     //如果在方法中要修改data的數據，可以通過this
     this.count +=1
   }
 }
 4.事件修飾符
 .prevent
 <a @click.prevent="xxx">連接</a>
 .stop
 <button @click.stop="xxx">按鈕</button>

5.條件渲染指令
 1.v-show的原理是：動態為元素添加或移除display:none來實現元素的顯示和隱藏
 --如果要頻繁的切換元素顯示，v-show性能會更好
 2.v-if的原理是：每次動態的創建或移除元素，實現元素的顯示和隱藏
 --如果剛進入頁面的時候，某些元素不需要被展示，且後期這個元素可能也不需要被展示，此時v-if性能更好
 3.在實際開發中絕大多數情況，不考慮性能問題，直接使用v-if
